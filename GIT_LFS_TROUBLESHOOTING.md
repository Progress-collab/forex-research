# Решение проблемы с Git LFS на Mac

## Проблема
Установка Git LFS зависла или не работает.

## Решения

### Вариант 1: Переустановить Git LFS

Если установка зависла, попробуйте:

```bash
# Прервать текущий процесс (Ctrl+C если еще запущен)

# Проверить установлен ли уже
which git-lfs
git-lfs version

# Если не установлен, попробуйте снова:
brew install git-lfs

# Или скачайте вручную с https://git-lfs.github.com/
```

### Вариант 2: Использовать без Git LFS (временно)

Если Git LFS не получается установить, можно добавить данные без LFS (но это не рекомендуется для больших файлов):

```bash
# Временно отключить LFS для этих файлов
git lfs untrack "*.parquet" 2>/dev/null || true

# Добавить данные обычным способом
git add data/v1/curated/ctrader/*.parquet

# Закоммитить
git commit -m "Add forex data files (without LFS)"

# Запушить
git push
```

⚠️ **Внимание**: Это может быть медленно для больших файлов!

### Вариант 3: Использовать GitHub Desktop

GitHub Desktop автоматически поддерживает Git LFS:
1. Установите GitHub Desktop: https://desktop.github.com/
2. Откройте проект в GitHub Desktop
3. Добавьте файлы через интерфейс
4. GitHub Desktop автоматически использует Git LFS если настроен

### Вариант 4: Загрузить данные позже

Можно продолжить работу без данных в git, а данные добавить позже:

```bash
# Просто закоммитить код без данных
git add .
git commit -m "Update project files"
git push

# Данные добавить позже когда Git LFS будет работать
```

## Рекомендация

Попробуйте **Вариант 1** - переустановить Git LFS. Если не получается - используйте **Вариант 4** (продолжить без данных в git пока).
