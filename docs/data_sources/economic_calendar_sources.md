# Источники экономического календаря для форекс-трейдинга

## Бесплатные источники

### 1. TradingEconomics API
- **URL**: https://tradingeconomics.com/api
- **Бесплатный тариф**: Ограниченный (до 500 запросов/месяц)
- **Формат**: JSON REST API
- **Данные**: Экономические события, индикаторы, календарь релизов
- **Регистрация**: Требуется API ключ (бесплатный)
- **Ограничения**: Rate limiting, ограниченная история
- **Рекомендация**: ✅ Хороший вариант для начала, но может потребоваться платный тариф для production

### 2. ForexFactory Calendar (парсинг)
- **URL**: https://www.forexfactory.com/calendar
- **Бесплатный тариф**: Да (через парсинг HTML)
- **Формат**: HTML страница
- **Данные**: Экономические события с фильтрами по важности
- **Регистрация**: Не требуется
- **Ограничения**: Нестабильный парсинг, возможны блокировки, нет официального API
- **Рекомендация**: ⚠️ Только для прототипирования, не для production

### 3. Investing.com Economic Calendar (парсинг)
- **URL**: https://www.investing.com/economic-calendar/
- **Бесплатный тариф**: Да (через парсинг)
- **Формат**: HTML страница
- **Данные**: Полный экономический календарь с фильтрами
- **Регистрация**: Не требуется
- **Ограничения**: Аналогично ForexFactory - нестабильный парсинг
- **Рекомендация**: ⚠️ Только для прототипирования

### 4. Alpha Vantage
- **URL**: https://www.alphavantage.co/
- **Бесплатный тариф**: Да (5 API calls/min, 500 calls/day)
- **Формат**: JSON REST API
- **Данные**: В основном рыночные данные, ограниченный экономический календарь
- **Регистрация**: Требуется бесплатный API ключ
- **Ограничения**: Rate limiting, фокус на рыночных данных, не на событиях
- **Рекомендация**: ❌ Не подходит для экономического календаря

### 5. FRED (Federal Reserve Economic Data)
- **URL**: https://fred.stlouisfed.org/docs/api/
- **Бесплатный тариф**: Да (без ограничений)
- **Формат**: JSON REST API
- **Данные**: Экономические индикаторы США, даты релизов
- **Регистрация**: Требуется бесплатный API ключ
- **Ограничения**: Только США, нет глобального календаря
- **Рекомендация**: ✅ Хорошо для данных по США, но недостаточно для полного календаря

## Платные источники (для справки)

### 1. Refinitiv (бывший Thomson Reuters)
- **Стоимость**: Высокая (корпоративные тарифы)
- **Данные**: Профессиональный экономический календарь
- **Рекомендация**: ❌ Слишком дорого для стартапа

### 2. Bloomberg API
- **Стоимость**: Очень высокая
- **Данные**: Полный доступ к экономическим данным
- **Рекомендация**: ❌ Не подходит для малых проектов

### 3. EODHistoricalData
- **Стоимость**: ~$20-50/месяц
- **Данные**: Экономический календарь + исторические данные
- **Рекомендация**: ⚠️ Рассмотреть при росте проекта

## Рекомендации для реализации

### Фаза 1 (MVP):
1. **TradingEconomics API** - начать с бесплатного тарифа
   - Регистрация API ключа
   - Интеграция через REST API
   - Кэширование данных локально
   - Ограничение: 500 запросов/месяц может быть недостаточно

### Фаза 2 (Production):
1. **Парсинг ForexFactory** как резервный источник
   - Использовать библиотеку BeautifulSoup/Scrapy
   - Реализовать retry logic и rate limiting
   - Кэшировать результаты агрессивно

2. **Комбинированный подход**:
   - TradingEconomics как основной источник
   - ForexFactory как резервный
   - FRED для данных по США

### Фаза 3 (Масштабирование):
1. Рассмотреть платные источники (EODHistoricalData) при росте нагрузки
2. Собственный парсер нескольких источников с агрегацией

## Структура данных для хранения

```json
{
  "event_id": "string",
  "country": "US",
  "event_name": "Non-Farm Payrolls",
  "importance": "high|medium|low",
  "actual": 150000,
  "forecast": 145000,
  "previous": 140000,
  "timestamp": "2025-11-08T12:30:00Z",
  "currency": "USD",
  "source": "tradingeconomics|forexfactory|fred"
}
```

## План интеграции

1. Создать модуль `src/data_pipeline/news_calendar.py`
2. Реализовать адаптеры для каждого источника
3. Создать агрегатор, объединяющий данные из разных источников
4. Сохранять в `data/v1/raw/news/economic_calendar.jsonl`
5. Обновлять через Prefect flow раз в день (или чаще перед важными событиями)
